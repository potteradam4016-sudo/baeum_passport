# 여행정보 초기화면 국가 추가/입력 구조 설계안

## Summary

여행정보 페이지는 대표 국가 중 하나를 고르는 기능이 아니라, 사용자가 여행정보를 작성하고 싶은 나라를 직접 추가하는 기능으로 구성한다.

현재 방향은 `/travel-info/[country]` 상세 페이지 중심이지만, 개편 후에는 `/travel-info`를 초기 화면으로 만들고 사용자가 직접 국가명을 입력한 뒤 해당 국가의 여행정보 작성 화면으로 이동한다.

- `/travel-info`: 여행정보 초기 화면, 국가 추가/목록 관리
- `/travel-info/[country]`: 사용자가 추가한 국가의 여행정보 작성 화면
- 초기 상태: 추가한 국가가 없으므로 `국가 추가` 버튼만 중심에 표시

## Key Changes

- 새 초기 화면 `/travel-info`를 추가한다.
- 여행정보 책갈피 링크는 특정 국가 상세가 아니라 `/travel-info`로 통일한다.
- 초기에는 추가된 국가 목록을 표시하지 않는다.
- `국가 추가` 버튼을 누르면 대표 국가 목록이 아니라 직접 국가명을 입력하는 화면 또는 모달을 보여준다.
- 사용자가 입력한 국가명으로 빈 여행정보 레코드를 생성하고 `/travel-info/[country]`로 이동한다.
- 사용자가 원하는 나라는 대표 국가 데이터에 없어도 추가할 수 있다.
- 기존 `/travel-info/[country]` 상세 작성 화면은 유지한다.

## Routing / Navigation

변경 후 흐름:

```text
여행정보 책갈피
↓
/travel-info
↓
국가 추가
↓
사용자가 국가명 직접 입력
↓
/travel-info/[country]
```

공통 책갈피의 여행정보 링크:

- 세계지도: `/travel-info`
- 사증: `/travel-info`
- 학습지: `/travel-info`
- 학습지 상세: `/travel-info`
- 여행정보 상세: `/travel-info`

특정 국가 상세 화면은 사용자가 직접 국가명을 입력해 추가하거나, 이미 추가된 국가 카드를 선택했을 때만 진입한다.

## Data Flow

기존 저장 구조인 `travelInfo: Record<string, TravelCountryInfo>`를 그대로 사용한다.

추가된 국가 판단:

```ts
Object.keys(state.travelInfo)
```

국가 추가 시 저장되는 기본값:

- `countryName`: 사용자가 입력한 국가명
- `displayName`: 사용자가 입력한 국가명
- `flagImage`: 빈 문자열
- `mapImage`: 빈 문자열
- `continent`: 빈 문자열
- 나머지 여행 준비 필드: 빈 문자열

이미 추가된 국가명을 다시 입력하면 새로 덮어쓰지 않고 기존 저장 내용을 유지한 채 `/travel-info/[country]`로 이동한다.

국가명 입력 처리:

- 앞뒤 공백은 제거한다.
- 빈 문자열은 추가하지 않는다.
- 같은 이름은 중복 추가하지 않는다.
- URL 경로에는 기존 `countryPath(name)`처럼 `encodeURIComponent` 기반 인코딩을 사용한다.

## Layout

### 1. 추가 국가가 없는 초기 화면

좌측 페이지:

- `Travel Info`
- `여행정보`
- “여행정보를 작성할 나라를 직접 추가해 보세요.”
- 여권 워터마크 장식

우측 페이지:

- 중앙에 `국가 추가` 버튼 하나만 표시
- 불필요한 빈 카드나 목록은 표시하지 않음

### 2. 국가 추가 입력 화면

좌측 페이지:

- `새 여행지 추가`
- “여행정보를 만들 나라 이름을 입력하세요.”
- 짧은 안내 문구

우측 페이지:

- 국가명 입력칸
- `추가하기` 버튼
- 입력 예시:
  - 이탈리아
  - 캐나다
  - 베트남
  - 스페인

### 3. 추가 국가가 있는 상태

좌측 페이지:

- `나의 여행정보`
- 작성한 여행정보 개수
- `국가 추가` 버튼

우측 페이지:

- 사용자가 추가한 국가 카드 목록
- 국가명 중심 표시
- 국기 이미지가 저장되어 있다면 작은 미리보기 표시
- 카드 클릭 시 `/travel-info/[country]`로 이동

## Test Plan

- `/travel-info` 진입 시 특정 국가 상세가 자동으로 열리지 않는지 확인
- 저장된 여행정보가 없을 때 `국가 추가` 버튼이 핵심 요소로 표시되는지 확인
- `국가 추가` 클릭 시 대표 국가 목록이 아니라 국가명 입력 UI가 보이는지 확인
- 빈 국가명은 추가되지 않는지 확인
- 국가명을 입력하고 추가하면 `/travel-info/[country]`로 이동하는지 확인
- 대표 국가 데이터에 없는 국가명도 추가 가능한지 확인
- 이미 추가된 국가명을 다시 입력하면 기존 상세 화면으로 이동하는지 확인
- 여행정보 작성 후 `/travel-info`로 돌아오면 해당 국가가 목록에 표시되는지 확인
- 모든 책갈피의 여행정보 링크가 `/travel-info`로 이동하는지 확인
- `/travel-info/[country]` 직접 접근은 계속 정상 동작하는지 확인
- 전체 화면 스크롤이 생기지 않는지 확인
- `npm.cmd run lint`, `npm.cmd run build` 통과 확인

## Assumptions

- “사용자가 원하는 나라를 입력”한다는 것은 대표 국가 목록에 없는 국가도 자유롭게 추가할 수 있다는 의미다.
- 여행정보 초기 화면은 사용자가 추가한 국가만 보여준다.
- 국가명은 사용자가 입력한 문자열을 기준으로 저장한다.
- 국기/지도 이미지는 상세 작성 화면에서 사용자가 URL로 입력한다.
- 기존 여행정보 상세 작성 화면의 펼쳐진 여권 레이아웃은 유지한다.
- 기존 `PROJECT_CONTEXT_RESTORE_REPORT.md`는 수정하지 않는다.
