# 배움여권 프로젝트 구현 분석 보고서

이 문서는 `README.md`를 프로젝트 요구사항 문서로 보고, 현재 코드베이스의 구현 상태를 대조한 분석 보고서입니다. 코드 수정은 수행하지 않았고, 현재 구현 현황과 누락 사항만 정리했습니다.

## 1. 현재 구현된 페이지 및 기능

### `/`

- 랜딩/홈 화면 구현
- 로그인, 회원가입 진입 버튼 제공
- 여권/도장 테마의 시각 요소 제공

### `/signup`

- 회원가입 폼 구현
- 입력 항목:
  - 학년
  - 반
  - 번호
  - 성
  - 이름
  - 생년월일
  - 성별
- 제출 시 `localStorage`에 사용자 일부 정보 저장
- 저장 후 `/login`으로 이동
- 관련 파일: `src/app/signup/page.tsx`

### `/login`

- 로그인 폼 구현
- 실제 인증 검증 없이 입력값 또는 더미 값으로 로그인 처리
- 로그인 후 `/worldmap`으로 이동
- 관련 파일: `src/app/login/page.tsx`

### `/worldmap`

- 대륙별 색상이 다른 단순 SVG 세계지도 구현
- 대륙 클릭 시 해당 대륙 정보 표시
  - 인구
  - 면적
  - 국가 수
  - 대표 국가
- 사이드탭 구현
  - 세계 여행
  - 방문 국가
- `세계 여행` 탭에서 대표국가 클릭 시 `/immi/[country]`로 이동
- `방문 국가` 탭에서 대표국가 클릭 시 `/workbook/[country]`로 이동
- 사용자가 국가명 입력 후 방문 국가 추가 가능
- 입국심사 완료 국가는 체크 아이콘으로 표시
- 관련 파일: `src/app/worldmap/page.tsx`

### `/immi/[country]`

- 대표국가 입국심사 페이지 구현
- 국가 정보와 인사말 표시
- 인사 따라쓰기 텍스트 입력 UI 제공
- 국기 색칠하기 형태의 3분할 색칠 UI 제공
- 완료 버튼 클릭 시 입국심사 완료 상태 저장
- 완료 후 `/worldmap`으로 이동
- 관련 파일: `src/app/immi/[country]/page.tsx`

### `/workbook/[country]`

- 대표국가 학습지 페이지 구현
- 표시 정보:
  - 국가 개요
  - 수도
  - 언어
  - 면적
  - 인구
  - 지도 설명
  - 전통 의상
  - 전통 음식
  - 전통 가옥
- 사용자 학습 내용 메모 작성 가능
- 작성 완료 시 학습 완료 상태 저장
- 완료 후 `/stamp`로 이동
- 관련 파일: `src/app/workbook/[country]/page.tsx`

### `/travel-info/[country]`

- 사용자가 추가한 여행국가 정보 입력 페이지 구현
- 입력 항목:
  - 국가명
  - 국기 이미지
  - 국가 이름
  - 면적
  - 인구 수
  - 사용 언어
  - 수도
  - 소속 대륙
  - 지도 이미지
- 저장 후 `/worldmap`으로 이동
- 관련 파일: `src/app/travel-info/[country]/page.tsx`

### `/stamp`

- 사증/도장 수집 페이지 구현
- 대표국가 전체 목록 표시
- 학습지가 완료된 국가는 도장 획득 상태로 표시
- 관련 파일: `src/app/stamp/page.tsx`

### 공통 구현

- `DashboardShell`: 공통 대시보드 레이아웃
- `Header`: 상단 헤더와 로그아웃 버튼
- `Sidebar`: 세계지도 페이지의 탭 전환 사이드바
- `WorldMapGraphic`: 클릭 가능한 SVG 세계지도
- `countries.ts`: 대륙 및 대표국가 데이터
- `storage.ts`: `localStorage` 기반 상태 저장

## 2. 아직 구현되지 않은 페이지 및 기능

- Spring Boot 백엔드
- PostgreSQL DB 연동
- 회원가입 API
- 로그인 API
- 로그아웃 API 또는 `/logout` 라우트
- 국가별 정보 백엔드 기능
- 사진 업로드
- 사진 삭제
- 실제 인증 검증
- 로그인/비로그인 접근 제어
- README 명세의 로그인 규칙
  - ID: `생년월일 + 성명`
  - PW: `생년월일`
- 생년월일, 성별의 실제 저장 및 활용
- 음성 인식 기반 인사 따라하기
- 실제 국기 이미지 기반 색칠하기
- 백지 국기 템플릿 제공
- 여행국가 정보의 국기 이미지/지도 이미지 실제 표시
- 대표국가 학습지의 개요 및 지도 수정 기능
- 사용자가 추가한 국가를 추가한 직후 여행국가 정보 페이지로 이동하는 흐름
- 서버 기반 영속성

## 3. 현재 라우팅 구조

현재 프로젝트는 Next.js App Router 구조입니다.

| 경로 | 파일 | 설명 |
| --- | --- | --- |
| `/` | `src/app/page.tsx` | 홈/랜딩 |
| `/signup` | `src/app/signup/page.tsx` | 회원가입 |
| `/login` | `src/app/login/page.tsx` | 로그인 |
| `/worldmap` | `src/app/worldmap/page.tsx` | 세계지도 |
| `/immi/[country]` | `src/app/immi/[country]/page.tsx` | 국가별 입국심사 |
| `/workbook/[country]` | `src/app/workbook/[country]/page.tsx` | 대표국가 학습지 |
| `/travel-info/[country]` | `src/app/travel-info/[country]/page.tsx` | 추가 여행국가 정보 입력 |
| `/stamp` | `src/app/stamp/page.tsx` | 사증/도장 페이지 |

README에는 `/travel_info`, `/immi`, `/workbook`처럼 적혀 있지만, 실제 구현은 국가명을 포함하는 동적 라우트입니다.

특히 여행국가 정보 페이지는 README의 `/travel_info`와 실제 구현의 `/travel-info/[country]` 표기가 다릅니다.

  - 학습지 메모

## 5. README.md 명세서 대비 누락된 기능

### 백엔드/DB

- Spring Boot 백엔드 없음
- PostgreSQL 연동 없음
- 회원가입, 로그인, 국가 정보 저장 API 없음
- 현재 모든 상태는 브라우저 `localStorage`에 저장됨

### 인증

- 실제 로그인 검증 없음
- README의 ID/PW 규칙 미구현
- `/logout` 라우트 없음
- 인증 상태에 따른 페이지 접근 제한 없음

### 회원가입

- 입력 UI는 있으나 일부 정보가 저장되지 않음
- 생년월일과 성별은 사용자 모델에 반영되지 않음
- 사진 업로드/삭제 없음

### 세계지도

- 대륙 클릭 및 정보 표시는 구현됨
- 대표국가 목록도 구현됨
- 방문 국가 추가는 가능함
- 다만 `+` 버튼 클릭 시 여행 국가 정보 페이지로 바로 이동하는 흐름은 미구현

### 여행국가 정보

- 입력 폼은 구현됨
- 실제 국기 이미지와 지도 이미지 표시 기능은 없음
- 이미지 URL 또는 설명을 텍스트로 입력하는 수준

### 입국심사

- 인사 따라하기와 국기 색칠하기 UI는 있음
- 두 기능 중 선택하는 구조는 아님
- 음성 인식 미구현
- 실제 국기 이미지와 백지 국기 제공 미구현

### 대표국가 학습지

- 국가 개요와 전통 요소 표시는 구현됨
- 사용자 메모는 가능함
- README의 “개요와 지도 수정 가능”은 미구현

### 사증

- 학습지 완료 국가의 도장 표시는 구현됨
- 추가 국가는 사증 생성 대상에서 제외됨
- 다만 이 조건을 명확히 분리한 정책/검증 코드는 부족함

## 6. 현재 UI/UX가 명세서와 다른 부분

- Full Screen Dashboard 요구는 대부분 반영되어 있음
  - `html`, `body`, root shell에 `height: 100vh`
  - 전역 `overflow: hidden`
  - 내부 스크롤은 `.scroll-area`로 제한

- `/`, `/login`, `/signup`은 대시보드라기보다 랜딩/폼 화면에 가까움

- 세계지도는 실제 지도 이미지가 아니라 단순화된 SVG 도형임

- 여권 테마는 반영되어 있음
  - 파란색 계열
  - 도장 요소
  - 종이 질감
  - 여권 느낌의 색상과 레이아웃

- 입국심사 페이지는 “인사 따라하기”와 “국기 색칠하기 중 선택”이 아니라 두 섹션이 동시에 노출됨

- 국기 색칠하기는 실제 국가 국기 형태가 아니라 3분할 색상 영역임

- 여행국가 정보 페이지는 이미지 입력값을 실제 이미지로 렌더링하지 않음

- 방문국가 탭의 `+` 버튼은 국가 추가만 수행하고, 여행국가 정보 페이지로 이동하지 않음

## 7. 다음 개발 우선순위 제안

### 1순위: 데이터와 인증 구조 결정

README 기준으로는 Spring Boot와 PostgreSQL이 핵심입니다.

우선 다음 중 하나를 결정해야 합니다.

- 현재처럼 프론트 프로토타입을 유지하면서 기능을 확장
- Spring Boot + PostgreSQL 기반으로 실제 서비스 구조 전환

명세 충족을 목표로 한다면 백엔드 API와 DB 연동을 가장 먼저 잡는 것이 좋습니다.

### 2순위: 회원가입/로그인 완성

- 생년월일, 성별 저장
- ID/PW 규칙 적용
- 로그인 검증 구현
- 로그아웃 구현
- 보호 라우트 적용

### 3순위: 라우팅 명세 통일

- README의 `/travel_info`와 실제 `/travel-info/[country]` 표기 정리
- `/logout` 처리 방식 확정
- 동적 라우트 사용 여부를 명세에 맞게 문서화

### 4순위: 세계지도 플로우 수정

- 방문국가 탭에서 `+` 버튼 클릭 후 국가 추가
- 추가 직후 `/travel-info/[country]`로 이동
- 대표국가와 추가국가의 이동 규칙 명확화

### 5순위: 입국심사 기능 고도화

- 인사 따라하기/국기 색칠하기 선택 모드 구현
- Web Speech API 등을 통한 음성 인식 검토
- 국가별 국기 템플릿 제공
- 실제 백지 국기 색칠 UX 개선

### 6순위: 여행국가 정보 및 사진 기능 구현

- 사진 업로드
- 사진 삭제
- 국기 이미지 미리보기
- 지도 이미지 미리보기
- 백엔드 저장 방식 확정

### 7순위: 학습지와 사증 UX 완성

- 대표국가 개요와 지도 수정 기능 추가
- 학습 완료 후 사증 생성 흐름 강화
- 추가 국가는 사증 생성 제외 조건을 명확히 코드로 분리

## 종합 평가

현재 프로젝트는 README의 프론트 화면 흐름을 빠르게 구현한 `localStorage` 기반 프로토타입에 가깝습니다.

페이지 뼈대, 대표국가 데이터, 세계지도, 입국심사, 학습지, 사증 페이지는 이미 존재합니다. 그러나 README에서 요구하는 핵심 시스템 요소인 Spring Boot 백엔드, PostgreSQL, 실제 인증, 사진 업로드/삭제, 음성 인식, 실제 이미지 기반 학습 기능은 아직 구현되지 않았습니다.

따라서 다음 단계에서는 단순 UI 추가보다 데이터 저장 구조와 사용자 인증 구조를 먼저 확정하는 것이 전체 개발 효율에 가장 중요합니다.
